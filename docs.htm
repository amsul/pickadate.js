pickadate.js

A lightweight jQuery dateinput picker

Weighs 6.9kb (2.8kb gzipped) – whereas jQuery UI datepicker is 30kb minified

Download pickadate.js or view it on Github

Simple setup

$( '.datepicker' ).pickadate()

Custom month and weekday tags

$( '.datepicker' ).pickadate({
    month_prev: '&larr;',
    month_next: '&rarr;',
    weekdays_short: [ 'Su', 'Mo', 'Tu', 'We', 'Th', 'Fr', 'Sa' ],
    show_months_full: false
})

Custom date formats

$( '.datepicker' ).pickadate({
    format: 'You selected: dddd, dd/mm/yyyy',
    format_submit: 'yyyy-mm-dd'
})

You can use format_submit to submit an alternate value to the server.

This is done by creating a hidden input field using the name of the original input plus the suffix _submit.

The date formats follow these rules:

Rule Result Example
d Date 1-31
dd Date with leading zero 01-31
ddd Weekday – short Sun-Sat
dddd Weekday – full Sunday-Saturday
m Month 1-12
mm Month with leading zero 01-12
mmm Month name – short Jan-Dec
mmmm Month name – full January-December
yy Year – short 00-99
yyyy Year – full 2000-2999

Disable picker for native calendar

$( '.datepicker' ).pickadate({
    disable_picker: true
})

Some browsers have good native support for type="date".

If you want to selectively disable the picker for these browsers, I would suggest detecing for support with something like Modernizr and then setting disable_picker to true.

Minimum and maximum date range

$( '.datepicker' ).pickadate({
    date_min: [ 2012, 10, 14 ],
    date_max: 5
})

date_min and date_max can be either

  1. an array (format: [ YYYY, MM, DD ] )
  2. or true / false

date_max can also be an integer - representing the number of days till the max date.

Arbitrary selectable dates

$( '.datepicker' ).pickadate({
    selectable_dates: [[2012,11,22],[2012,11,23],[2012,11,26]]
})

selectable_dates is an array of arbitrary selectable dates. All other are disabled.

Set Monday as the first day

$( '.datepicker' ).pickadate({
    first_day: 1
})

Set first_day to 1 for Monday and 0 for Sunday.
